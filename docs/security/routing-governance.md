# Routing governance

Routing contracts have no credential, API-key, token, password, endpoint-secret, or provider
configuration fields. Unknown fields fail validation. The router cannot enable providers, change
base URLs, alter retries or timeouts, grant network access, install SDKs, expand Controller budgets,
bypass Context validation, or treat provider prose as measured evidence.

Static routing stays available. Shadow decisions cannot execute. Adaptive policies require a
verified promotion assessment, explicit operator approval, and bounded TaskSignature scope.
PostgreSQL protects revisions, observations, decisions, outcomes, experiments, and access records
with append-only triggers and exposes profile and policy transitions through controlled functions.
Security tests cover provider self-promotion, health bypass, Context bypass, shadow interference,
uncertain-side-effect fallback, and budget expansion.
