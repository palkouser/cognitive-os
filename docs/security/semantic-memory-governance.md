# Semantic-memory governance

PostgreSQL is authoritative for observations, claims, evidence, relations, contradictions, Wiki
projections, and semantic access audit. Existing events remain lifecycle evidence, artifacts remain
large-content authority, and Sprint 9 memory remains source-memory authority. Providers receive no
database credentials or semantic write tool.

Provider output is proposal-only. Direct provider commits, unknown predicates, fabricated source
references, confidence-only promotion, automatic supported promotion, graph databases, hybrid
retrieval, generated Wiki narrative, and Wiki-as-evidence all fail closed. Supported promotion
requires exact revision evidence, all confidence dimensions, a trusted actor, a registered
verifier-bundle decision, and no deterministic unresolved conflict.

Queries enforce scope, sensitivity, and result budgets and append an access row for every returned
claim. Access rows contain IDs, revision, mode, times, rank, scope, sensitivity, and hashes, never
claim text. Markdown and HTML-like input is escaped as data. Append-only triggers and runtime
grants prevent history updates or deletes. Health diagnostics report event/projection divergence
without destructive repair.

NetworkX 3.x is an optional BSD-3-Clause analytical dependency under the `semantic-graph` extra.
It receives only bounded exact revision edges, executes no untrusted code, and owns no state. The
stdlib traversal remains the core path, so removing the extra does not affect persistence.
