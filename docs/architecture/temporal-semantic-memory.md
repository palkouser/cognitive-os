# Temporal semantic memory

Sprint 10 turns explicitly selected, authoritative Sprint 9 revisions into immutable semantic
observations and evidence-backed claims. Observations preserve exact source hashes and field, line,
or byte grounding. Providers may return bounded proposals but have no repository or promotion
authority.

Claims use stable subject/predicate identity and append-only bitemporal revisions. Host-owned frozen
predicate descriptors define value types, cardinality, and bounded contradiction rules. Supported
promotion requires complete confidence dimensions, exact evidence, registered verification, a
trusted actor, and no deterministic unresolved conflict.

PostgreSQL is authoritative. Lifecycle events provide replay and consistency evidence. Relational
edges are the graph; optional in-process analysis is only a bounded projection. Wiki v3 is
deterministic Markdown with exact claim-revision lineage and is never evidence.

Operations use migration `0003`, `scripts/semantic_health.py`, isolated backup/restore, and
credential-free smoke and benchmark fixtures. Sprint 11 may consume read contracts but must not
replace claim history or automatically inject it into provider context.
