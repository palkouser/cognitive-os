# Problem Representation

`ProblemRepresentation` is the canonical execution input. Deterministic normalization creates
a stable request hash, artifact references, registry summaries, and non-removable policy
constraints. Structured provider output is schema-validated, merged below machine policy, and
persisted as an append-only revision. A representation is executable only when it has goals,
outputs, required acceptance criteria, no hard conflict, and no unanswered required question.
