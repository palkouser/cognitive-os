# Weakness Mining Engine

Sprint 17 adds a diagnostic-only engine over existing Controller, Context, provider, Tool Plane,
verifier, acceptance, memory, semantic, skill, strategy, experience, corpus, routing, event, and
artifact boundaries. Read-only resolvers freeze exact source revisions. Registered extractors emit
typed signals; exact signatures produce authoritative groups; optional clusters remain advisory.

The deterministic pipeline snapshots sources, extracts and validates signals, groups, scores 13
impact dimensions, packages evidence, creates candidate-only weakness revisions, orders eligible
items in the Weakness Queue, records accesses, and seals a replay manifest. PostgreSQL migration
`0009` owns ten tables, eight append-only histories, current-item projections, and controlled
lifecycle and queue functions. The engine cannot confirm findings, execute fixes, alter source
systems, promote candidates, or change Controller/provider/tool policy.

Identity and replay are content-addressed. A stable weakness ID derives from its signature, while
each source, signal set, group/cluster snapshot, score, package, revision, queue snapshot, and run
manifest carries its own hash. No-op clustering keeps core installation CPU-only and dependency
light.
